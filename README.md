# Word AI 添削アプリ (Word AI Proofreader App)

Wordファイルの文章をAI（社内専用AI、OpenAI API等）を使って自動的に添削し、**変更履歴（校閲モード）**と**コメント**を付与した状態で出力するデスクトップアプリケーションです。

最新のアップデートにより、**「方法4：マルチエージェント（マルチパス）方式」**を導入し、技術報告書等の高度な添削に対応しました。

## 主な機能

### 1. 高度なマルチパス添削 (New!)
単純なチャンク分割だけでなく、以下の3つの役割（パス）を順番に実行することで、文章全体の質を向上させます。
- **全体構成・論理レビュー**: 技術報告書としての構成評価、論理の矛盾や飛躍の指摘。
- **用語統一・表記揺れチェック**: 「サーバ/サーバー」等の表記揺れを自動検出し、文書全体で統一。
- **詳細校正**: 誤字脱字、文法的誤り、てにをはの精査。

### 2. 多彩なAI接続対応
- **カスタムエンドポイント対応**: 社内GPTサーバーやローカルのAI（LM Studio等）を自由に設定可能。
- **gpt-4o-mini 最適化**: 128kの広大なコンテキストウィンドウを活かし、構成レビューや用語抽出では文章全体を一度に解析します。

### 3. Word 連携
- **変更履歴の自動記録**: 修正箇所をWordの「変更履歴の記録」機能を使って反映。
- **修正理由のコメント挿入**: なぜそのように修正したのか、理由をWordのコメントとして挿入。

## セットアップ

### 1. 準備
- Python 3.10 以上がインストールされている必要があります。
- 接続先のAIエンドポイント（社内AIサーバー、OpenAI API、または LM Studio等）のURLとAPIキーが必要です。

### 2. 依存ライブラリのインストール
```bash
pip install -r requirements.txt
```

## 使い方

1. アプリケーションを起動します。
   ```bash
   python main.py
   ```
2. 設定（Config）にて、**AIエンドポイントURL** と **APIキー** を入力します。
3. 必要に応じて **システムプロンプト** を調整します（技術報告書用、メール用など）。
4. 「参照」ボタンから、添削したい **Wordファイル (.docx)** を選択します。
5. 「添削を開始する」ボタンをクリックします。
6. 処理が完了すると、元のファイル名に `_添削済み` が付与された新しいファイルが保存されます。

## 開発と検証
- 本ツールの高度化にあたっては、LM Studio による設計支援および自動テスト（`test_custom_ai.py`）を活用しています。

## 注意事項
- 処理対象は `.docx` 形式のファイルのみです。
- 長文の全文解析など、APIのタイムアウト設定にご注意ください（現状、デフォルトで120秒以上に設定されています）。
